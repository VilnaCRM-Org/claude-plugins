# Project Profile Schema — `.claude/php-sdlc.yml`

The project profile is the single source of truth that generalizes every
command, agent, and skill in this plugin to a concrete PHP backend
repository (FR-17). It lives at `.claude/php-sdlc.yml` in the target
repository, is generated by `/sdlc-setup` (`scripts/generate-profile.sh`),
and is validated by `scripts/validate-profile.sh` before every SDLC stage.

This document is the **canonical key list**: the `profile-keys-check` CI
job greps each skill's `## Profile keys consumed` header against the keys
documented here. A key consumed by a skill but absent from this page fails
CI.

## Top-level keys

| Key | Required | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `schema_version` | yes | int | — | Must be `1`. Bumped only on breaking schema changes (plugin MAJOR release). |

## `project`

| Key | Required | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `project.name` | yes | string | — | Short service name; used in reports and issue/PR text. |
| `project.repo` | yes | string | — | GitHub `owner/name`; consumed by `gh` calls. |

## `php`

| Key | Required | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `php.version` | yes | string | — | Detected from `composer.json` (`require.php`). Quote it: `"8.4"`. |

## `framework`

| Key | Required | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `framework.name` | yes | string | — | e.g. `symfony`. |
| `framework.version` | no | string | — | Informational. |
| `framework.api_platform` | no | `false` \| version string | `false` | Version string when API Platform is installed. |
| `framework.graphql` | no | bool | `false` | True when a GraphQL layer is detected. |

## `persistence`

| Key | Required | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `persistence.mapper` | yes | enum | — | `doctrine-orm` \| `doctrine-odm`. |
| `persistence.engine` | yes | enum | — | `mysql` \| `mariadb` \| `postgresql` \| `mongodb`. |

## `architecture`

| Key | Required | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `architecture.source_root` | yes | string | — | Usually `src`. |
| `architecture.bounded_contexts` | yes | list | — | At least one context directory name under the source root. |
| `architecture.shared_context` | no | string \| null | `null` | Shared-kernel context name; `null` = none. |

## `make` — logical target map

The `make` map is **required and must be complete**: every key below must
be declared. The value is the actual Make target name in the target
repository. An explicit `null` value means **capability absent** (NFR-4):
the plugin skips or degrades the dependent behavior with a note instead of
failing. Omitting a key entirely is a validation error — the distinction
between "absent capability" and "forgot to declare" is intentional.

| Key | Required | Default target | Capability |
| --- | --- | --- | --- |
| `make.ci` | yes (nullable) | `ci` | Full local CI suite. |
| `make.start` | yes (nullable) | `start` | Boot the service (containers). |
| `make.tests` | yes (nullable) | `tests` | Unit/integration tests. |
| `make.e2e` | yes (nullable) | `e2e-tests` | End-to-end tests. |
| `make.psalm` | yes (nullable) | `psalm` | Static analysis. |
| `make.deptrac` | yes (nullable) | `deptrac` | Layer-dependency check. |
| `make.phpinsights` | yes (nullable) | `phpinsights` | Code-quality scores. |
| `make.infection` | yes (nullable) | `infection` | Mutation testing. |
| `make.ai_review_loop` | yes (nullable) | `null` | AI review loop; plugin substitutes `scripts/ai-review-loop.sh` when `null`. |
| `make.pr_comments` | yes (nullable) | `null` | PR comment listing; plugin substitutes `scripts/get-pr-comments.sh` when `null`. |
| `make.fr_nfr_gate` | yes (nullable) | `null` | FR/NFR gate; plugin substitutes `scripts/fr-nfr-gate.sh` when `null`. |
| `make.load_tests` | yes (nullable) | `load-tests` | Load tests; pair with `capabilities.load_testing`. |
| `make.security` | yes (nullable) | `null` | Security/SAST suite; plugin runs its bundled static lane (Psalm taint / Semgrep / `composer audit` / secret-scan) when `null` (SA-2). Mirrors `make.ai_review_loop`/`make.pr_comments`/`make.fr_nfr_gate` null-substitution precedent. |
| `make.post_review_findings` | yes (nullable) | `null` | PR-comment publisher; plugin substitutes `scripts/post-review-findings.sh` when `null`. Mirrors `make.ai_review_loop`/`make.pr_comments`/`make.fr_nfr_gate` null-substitution precedent. |

## `quality` — raise-only thresholds (ADR-7)

Shipped defaults are the **minimum bar**. The `quality.phpinsights` map
holds the four phpinsights score thresholds; the remaining keys are
top-level counters and scores. A profile may tighten the bar, never relax
it:

- Score thresholds (`quality.phpinsights.*`, `quality.infection_msi`) may
  be **raised** above the defaults, never lowered.
- Violation-count ceilings (`quality.deptrac_violations`,
  `quality.psalm_errors`) ship at `0` and may not be raised — allowing
  more violations would lower the bar.

`validate-profile.sh` rejects any value on the wrong side of its default.

| Key | Required | Direction | Shipped default |
| --- | --- | --- | --- |
| `quality.phpinsights.quality` | yes | floor (raise-only) | `100` |
| `quality.phpinsights.architecture` | yes | floor (raise-only) | `100` |
| `quality.phpinsights.style` | yes | floor (raise-only) | `100` |
| `quality.phpinsights.complexity` | yes | floor (raise-only) | `94` |
| `quality.deptrac_violations` | yes | ceiling (fixed) | `0` |
| `quality.psalm_errors` | yes | ceiling (fixed) | `0` |
| `quality.infection_msi` | yes | floor (raise-only) | `100` |

## `ci`

| Key | Required | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `ci.provider` | yes (nullable) | string \| null | — | `github-actions`, or explicit `null` = no CI: `/sdlc-finish-pr` skips check-waiting with a degrade note (NFR-4). The key itself must be declared. |
| `ci.workflows` | no | list | `[]` | Workflow names detected in the repository. |
| `ci.required_checks` | no | list | `[]` | Check names that must be green before a PR is finished. |

## `review`

| Key | Required | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `review.coderabbit` | no | bool | `false` | True when `.coderabbit.yaml` exists; when false, `/sdlc-finish-pr` uses the AI review loop as its comment source (NFR-4). |
| `review.ai_review_agents` | no | list | `[claude]` | v1 supports `claude` only; other entries warn-and-skip (ADR-8). |
| `review.request_changes_blocking` | no | bool | `true` | Whether a CHANGES_REQUESTED review blocks finishing. |

## `capabilities`

| Key | Required | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `capabilities.structurizr` | no | bool | `false` | Gates the architecture-sync skill (skip-with-note when false). |
| `capabilities.observability_emf` | no | bool | `false` | Selects the metrics emission backend for the observability skill: AWS EMF when `true`, generic backend (Prometheus/StatsD/OTel/structured logs) when `false`. Never skip-gates the skill. |
| `capabilities.load_testing` | no | bool | `false` | Gates load-test skills; pair with `make.load_tests`. |
| `capabilities.dynamic_security_testing` | no | bool | `false` | Gates **dynamic** (live-service) security probing in `security-audit`; pairs with `make.start` the way `capabilities.load_testing` pairs with `make.load_tests`. When `false` (or `make.start: null`), dynamic probing degrades to skip-with-note; static/SAST/dep/secret/config lanes still run (NFR-3). |
| `capabilities.publish_pr_comments` | no | bool | `false` | Gates the Publish step + the poster's write path (default OFF / opt-in); when false/absent the Publish step and the poster skip-with-note (NFR-3). Mirrors `capabilities.dynamic_security_testing` / `capabilities.structurizr`. |

## Annotated example

Complete profile with the canonical reference values from the upstream
reference service. The fence marker keeps this block out of the NFR-2
generalization audit:

```yaml # profile-example
schema_version: 1                  # required; int; bump on breaking schema change
project:
  name: user-service               # required
  repo: VilnaCRM-Org/user-service  # required, owner/name
php:
  version: "8.4"                   # required
framework:
  name: symfony                    # required
  version: "7.3"                   # optional
  api_platform: "4.2"              # false | version string; default false
  graphql: true                    # bool; default false
persistence:
  mapper: doctrine-odm             # required: doctrine-orm | doctrine-odm
  engine: mongodb                  # required: mysql | mariadb | postgresql | mongodb
architecture:
  source_root: src                 # required
  bounded_contexts: [User, OAuth]  # required, >=1
  shared_context: Shared           # optional; null = none
make:                              # required map; null value = capability absent (NFR-4)
  ci: ci
  start: start
  tests: tests
  e2e: e2e-tests
  psalm: psalm
  deptrac: deptrac
  phpinsights: phpinsights
  infection: infection
  ai_review_loop: null             # plugin script substitutes when null
  pr_comments: null
  fr_nfr_gate: null
  load_tests: load-tests
  security: null                   # plugin runs bundled static lane when null
  post_review_findings: null       # plugin substitutes scripts/post-review-findings.sh when null
quality:                           # required; values may only be RAISED vs these defaults
  phpinsights: {quality: 100, architecture: 100, style: 100, complexity: 94}
  deptrac_violations: 0
  psalm_errors: 0
  infection_msi: 100
ci:
  provider: github-actions         # required; null = no CI (degrade)
  workflows: [tests, psalm, deptrac, phpinsights, infection, openapi-validate]
  required_checks: [tests, psalm, deptrac]
review:
  coderabbit: true                 # bool
  ai_review_agents: [claude]       # default [claude]; non-claude entries warn+skip in v1
  request_changes_blocking: true   # bool
capabilities:
  structurizr: true                # bool, default false
  observability_emf: true
  load_testing: true
  dynamic_security_testing: false  # gates live-service probing; static lanes run regardless
  publish_pr_comments: false       # bool, default false; opt-in PR-comment publishing
```

## Validation

```bash
"${CLAUDE_PLUGIN_ROOT}/scripts/validate-profile.sh" [PROFILE_FILE]
```

Defaults to `<cwd>/.claude/php-sdlc.yml`. Prints one `VIOLATION:` line per
finding (all findings, not just the first) and exits `0` when valid, `1`
otherwise. Violation classes: missing/undeclared required key, illegal
enum value, threshold on the wrong side of its shipped default,
unsupported `schema_version`, incomplete `make` map.

## For skill authors

Reference profile keys as inline code with the full dotted path (e.g.
`persistence.mapper`, `make.ci`) and list every consumed key under a
`## Profile keys consumed` heading at the top of the skill body. CI greps
that list against this page; an undeclared key fails the build. Concrete
upstream values may appear only inside fenced blocks whose opening fence
carries the `# profile-example` marker.
